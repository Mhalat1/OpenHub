💾 Runbook: Mémoire PHP

Alerte: Utilisation mémoire PHP élevée
Seuil: > 200 MB
Instance: openhub-backend.onrender.com

📋 Description

L'utilisation mémoire PHP est anormalement élevée, ce qui peut précéder un crash "out of memory".

🔍 Diagnostic

1. Identifier les gros consommateurs

// Dans le code, logger la mémoire
error_log("Peak: " . memory_get_peak_usage(true) / 1048576 . " MB");
error_log("Current: " . memory_get_usage(true) / 1048576 . " MB");

2. Analyser par endpoint

# Voir quelles routes consomment le plus
# Middleware de logging mémoire
// Avant traitement
$startMemory = memory_get_usage();
// Après traitement
$endMemory = memory_get_usage();
log("Route: " . $request->path() . " Memory: " . ($endMemory - $startMemory) / 1048576 . " MB");

3. Vérifier les fuites mémoire

# Si mémoire augmente avec le temps
# Voir les processus PHP qui vieillissent
# pm.max_requests = 500 (redémarrage périodique)

🛠️ Optimisations

Libérer la mémoire

// Unset grandes variables après usage
unset($largeArray);
gc_collect_cycles();

// Traitement par lots
foreach (array_chunk($hugeDataset, 100) as $chunk) {
    processChunk($chunk);
}

Optimiser les requêtes

// Sélectionner uniquement les colonnes nécessaires
User::select('id', 'name', 'email')->get();

// Paginer
$users = User::paginate(50);

Configuration PHP

# php.ini
memory_limit = 256M  # Augmenter si nécessaire
max_execution_time = 30
max_input_time = 60
