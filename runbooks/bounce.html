🔄 Runbook: Bounce / Instabilité

Alerte: Redémarrages répétés détectés
Sévérité: MAJEUR - P2
Instance: open_hub-backend.onrender.com

📋 Description

L'application redémarre de façon répétée (bounce). Cela indique une instabilité critique qui peut être causée par :

  • Crash immédiat au démarrage
  • Manque de ressources récurrent
  • Base de données inaccessible au démarrage
  • Configuration invalide

⚡ Actions immédiates

1. Examiner les logs de démarrage

# Logs récents (derniers redémarrages)
tail -100 /var/log/php.log
tail -100 /var/log/php-fpm.log

# Chercher des erreurs systématiques
grep -i "error\|fatal\|exception" /var/log/php.log | tail -20

2. Vérifier les ressources

# Utilisation mémoire avant crash
# Via Render Dashboard → Metrics

# Limites Render actuelles
# Starter: 512 MB RAM
# Professional: 1-2 GB RAM

3. Tester en isolation

# Désactiver temporairement les workers
# Vérifier si le problème persiste sans charge

# Tester avec une configuration minimale
php -S localhost:8080 -t public/  # Serveur de développement

🔍 Causes et solutions

Cause Symptômes Solution
🧠 Out of Memory "Killed", "Out of memory" dans logs Augmenter memory_limit ou scale up
🗄️ DB timeout au démarrage "Connection refused" dans logs Vérifier DB, augmenter timeout
📦 Extension PHP manquante "Class not found", "Function undefined" Installer extension requise
⚙️ .env invalide Erreurs de configuration Vérifier variables d'environnement
🔁 Loop de redémarrage Redémarrage toutes les 10-30s Rollback immédiat

🛠️ Solutions

Augmenter les limites mémoire

# Dans php.ini
memory_limit = 256M  # Augmenter à 512M ou 1G

# Redémarrer PHP-FPM
sudo service php8.1-fpm restart

Augmenter les timeouts

# Dans config/database.php (Laravel)
'options' => [
    PDO::ATTR_TIMEOUT => 30,  # Augmenter à 60
],

# Dans php.ini
max_execution_time = 30  # Augmenter à 60
max_input_time = 60

Rollback préventif

# Si déploiement récent, rollback immédiat
1. Render Dashboard → Deployments
2. Rollback à version stable
3. Analyser différence plus tard
