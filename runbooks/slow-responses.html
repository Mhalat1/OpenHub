🐢 Runbook: Réponses lentes (P95 > 1s)

Alerte: Temps de réponse P95 > 1 seconde
Instance: open_hub-backend.onrender.com

📋 Description

Le 95ème percentile des temps de réponse dépasse 1 seconde. Cela signifie que 5% des requêtes prennent plus d'une seconde, ce qui dégrade l'expérience utilisateur.

🔍 Identifier les requêtes lentes

1. Analyser les logs d'accès

# Trouver les requêtes lentes dans nginx
awk '{if ($NF > 1) print $0}' /var/log/nginx/access.log

# Trier par temps de réponse
awk '{print $NF, $7}' /var/log/nginx/access.log | sort -rn | head -20

# Avec format de log incluant temps
log_format timed '$remote_addr - $remote_user [$time_local] '
                 '"$request" $status $body_bytes_sent '
                 '$request_time $upstream_response_time';

2. Activer slow query log

# MySQL
SET GLOBAL slow_query_log = 'ON';
SET GLOBAL long_query_time = 1;  # Requêtes > 1 seconde

# PostgreSQL
# Dans postgresql.conf
log_min_duration_statement = 1000  # millisecondes

3. Profiler le code PHP

// Utiliser Laravel Debugbar en dev
// Ou ajouter du logging
$start = microtime(true);
// code à mesurer
$time = microtime(true) - $start;
Log::channel('slow')->warning('Slow operation', [
    'time' => $time,
    'route' => request()->path()
]);

⚡ Causes fréquentes

Cause Indice Solution
Requêtes N+1 Nombreuses requêtes DB Eager loading
Index manquants Full table scans Ajouter indexes
Données volumineuses SELECT sans LIMIT Pagination
Cache non utilisé Mêmes données chargées Mettre en cache
API externe lente Temps passé en HTTP Cache ou async

🛠️ Optimisations

Eager Loading (Laravel)

// Mauvais (N+1)
$posts = Post::all();
foreach($posts as $post) {
    echo $post->user->name;
}

// Bon (eager loading)
$posts = Post::with('user')->get();

Ajouter des indexes

-- Trouver les requêtes lentes
-- Ajouter des indexes
CREATE INDEX idx_users_email ON users(email);
CREATE INDEX idx_posts_user_id ON posts(user_id);

Mettre en cache

// Cache Redis/Memcached
$users = Cache::remember('active_users', 3600, function() {
    return User::where('active', 1)->get();
});

Pagination

// Au lieu de all()
$posts = Post::paginate(20);

// Avec curseur pour gros volumes
foreach (Post::cursor() as $post) {
    // Traitement par lot
}

Optimiser les appels API

// Mettre en cache les réponses externes
$data = Cache::remember('api_data', 300, function() {
    return Http::get('https://api.externe.com/data')->json();
});
