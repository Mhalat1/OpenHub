🔥 Runbook: Erreurs HTTP 5xx

CRITIQUE - P1
Alerte: Erreurs HTTP 5xx détectées
Instance: open_hub-backend.onrender.com
Action: Intervention requise immédiatement

📋 Description

Des erreurs serveur (500-599) sont détectées. Cela indique un problème côté serveur qui empêche le traitement correct des requêtes. Impact direct sur les utilisateurs.

🔍 Analyse des codes

Code Signification Cause probable
500 Internal Server Error Erreur générique Exception PHP non catchée, erreur fatale
502 Bad Gateway Proxy reçoit réponse invalide PHP-FPM crashé ou timeout
503 Service Unavailable Service temporairement indispo Maintenance, surcharge
504 Gateway Timeout Timeout proxy Requête trop longue

⚡ Actions immédiates

1. Vérifier les logs PHP

# Logs d'erreur PHP
tail -100 /var/log/php.log
tail -100 /var/log/php-fpm.log

# Chercher les erreurs fatales
grep -i "fatal\|error\|exception" /var/log/php.log -B5 -A5

2. Vérifier les logs accès

# Voir quelles URLs génèrent des 5xx
grep " 5[0-9][0-9] " /var/log/nginx/access.log | awk '{print $7}' | sort | uniq -c | sort -rn

# Voir par IP
grep " 500 " /var/log/nginx/access.log | awk '{print $1}' | sort | uniq -c | sort -rn

3. Vérifier l'état des services

# Statut PHP-FPM
sudo service php8.1-fpm status
sudo systemctl status php-fpm

# Vérifier base de données
mysqladmin ping
pg_isready

🛠️ Résolution par type

500 - Erreur PHP

# Activer l'affichage temporaire des erreurs
// Dans index.php
ini_set('display_errors', 1);
error_reporting(E_ALL);

# Voir la stack trace dans les logs
# Corriger le bug identifié

502 - Bad Gateway

# PHP-FPM est probablement crashé
sudo service php8.1-fpm restart

# Vérifier les logs avant crash
# Augmenter les timeouts si nécessaire

503 - Service Unavailable

# Vérifier si mode maintenance activé
# Désactiver si accidentel
php artisan up  # Laravel

# Vérifier les ressources
# Si surcharge, augmenter workers

504 - Gateway Timeout

# Augmenter les timeouts
# Dans nginx config
proxy_read_timeout 60s;
fastcgi_read_timeout 60s;

# Optimiser les requêtes lentes

📊 Vérification post-résolution

# Tester les endpoints problématiques
curl -I https://open_hub-backend.onrender.com/endpoint-probleme

# Vérifier que les 5xx diminuent
tail -f /var/log/nginx/access.log | grep " 5[0-9][0-9] "

# Vérifier Grafana
# Le taux d'erreur doit revenir à 0
