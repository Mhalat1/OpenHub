🚫 Runbook: Erreurs HTTP 4xx

Alerte: Spike d'erreurs HTTP 4xx
Instance: open-hub-backend.onrender.com

📋 Description

Augmentation soudaine des erreurs 4xx (400-499). Ces erreurs sont généralement côté client, mais peuvent indiquer des problèmes d'API ou de configuration.

🔍 Analyse des codes

Code Signification Cause probable
400 Bad Request Requête malformée Client envoie mauvais format
401 Unauthorized Authentification manquante Token expiré/manquant
403 Forbidden Accès interdit Permissions insuffisantes
404 Not Found Ressource inexistante URL changée ou supprimée
405 Method Not Allowed Méthode HTTP invalide Client utilise mauvaise méthode
409 Conflict Conflit avec état actuel Version obsolète, doublon
422 Unprocessable Entity Validation échouée Données invalides
429 Too Many Requests Rate limit dépassé Client trop rapide

⚡ Actions immédiates

1. Analyser les logs

# Voir distribution des 4xx
grep " 4[0-9][0-9] " /var/log/nginx/access.log | cut -d' ' -f9 | sort | uniq -c | sort -rn

# Voir les URLs problématiques
grep " 404 " /var/log/nginx/access.log | awk '{print $7}' | sort | uniq -c | sort -rn | head -10

2. Identifier le pattern

# Par IP source
grep " 401 " /var/log/nginx/access.log | awk '{print $1}' | sort | uniq -c | sort -rn

# Par user-agent (bot ou humain ?)
grep " 429 " /var/log/nginx/access.log | awk -F'"' '{print $6}' | sort | uniq -c | sort -rn

3. Vérifier les changements récents

# Déploiements récents
git log --oneline -10

# Changements d'API
git diff HEAD~10 -- routes/

🛠️ Résolution par code

Pour 400/422 (Validation)

// Vérifier la validation des requêtes
// Laravel
php artisan route:list
// Voir les règles de validation dans FormRequests

Pour 401/403 (Auth)

# Vérifier le service d'authentification
# Token JWT expiré ? Clé API changée ?
# Vérifier la configuration CORS

Pour 404 (Routes)

# Vérifier que les routes existent
php artisan route:list | grep [endpoint]

# Vérifier les redirections
# Si des URLs ont été changées, ajouter des redirections

Pour 405 (Methods)

# Vérifier les méthodes autorisées
# Dans les routes
Route::match(['GET', 'POST'], '/endpoint', ...);

Pour 429 (Rate limit)

# Ajuster les rate limits temporairement
# Dans Laravel (Kernel.php)
'throttle:60,1'  # Augmenter à 100,1

# Ou contacter le client concerné

📝 Post-incident

  • Si changement d'API → Mettre à jour la documentation
  • Si bug → Corriger et déployer
  • Si attaque (bruteforce) → Renforcer sécurité
  • Ajouter monitoring sur endpoints sensibles
