⚡ Runbook: Charge CPU

Alerte: Charge CPU moyenne élevée
Condition: load > 5 sur 1 minute
Instance: openhub-backend.onrender.com

📋 Description

La charge CPU dépasse le seuil critique. Cela peut entraîner des lenteurs, des timeouts, et une dégradation de l'expérience utilisateur.

⚡ Actions immédiates

1. Identifier les processus gourmands

# Via Render Dashboard
https://dashboard.render.com/web/openhub-backend/metrics

# Top processus (si accès SSH)
top -b -n 1 | head -20
ps aux --sort=-%cpu | head -10

2. Analyser les requêtes PHP lentes

# Activer slow log temporairement
# Dans php.ini
slowlog = /var/log/php-slow.log
request_slowlog_timeout = 5s

# Redémarrer PHP-FPM
sudo service php8.1-fpm restart

# Analyser
tail -f /var/log/php-slow.log

3. Vérifier les requêtes DB

# MySQL
SHOW FULL PROCESSLIST;
SHOW STATUS LIKE 'Threads_connected';

# PostgreSQL
SELECT * FROM pg_stat_activity WHERE state = 'active';

🔍 Diagnostic

Symptôme Cause possible Action
CPU utilisateur élevé Code PHP inefficace Optimiser algorithmes
CPU système élevé I/O, appels système Vérifier accès disque/network
Nombreux processus Trop de workers PHP Ajuster pm.max_children
Requêtes DB lentes Index manquants Ajouter indexes

🛠️ Solutions

Ajuster PHP-FPM

# /etc/php/8.1/fpm/pool.d/www.conf
pm = dynamic
pm.max_children = 5  # Réduire si CPU limité
pm.start_servers = 2
pm.min_spare_servers = 1
pm.max_spare_servers = 3

Optimiser les requêtes

-- Ajouter des indexes
CREATE INDEX idx_users_email ON users(email);

-- Analyser les requêtes lentes
EXPLAIN SELECT * FROM large_table WHERE condition;

Mettre en cache

// Redis/Memcached
$users = cache()->remember('active_users', 3600, function() {
    return DB::table('users')->where('active', 1)->get();
});
