🟨 Runbook: Erreurs JavaScript

Alerte: Erreurs JavaScript côté client détectées
Impact: Fonctionnalités frontend cassées

📋 Description

Des erreurs JavaScript sont rapportées par les navigateurs des utilisateurs. Cela peut casser l'interface utilisateur et les interactions.

🔍 Investigation

1. Capturer les erreurs

// Dans la console navigateur
// Ou via Sentry/LogRocket

// Ajouter un logger global
window.addEventListener('error', (event) => {
    fetch('/api/log-js-error', {
        method: 'POST',
        body: JSON.stringify({
            message: event.message,
            filename: event.filename,
            lineno: event.lineno,
            colno: event.colno,
            stack: event.error?.stack
        })
    });
});

2. Types d'erreurs courantes

  • TypeError - Variable undefined, fonction inexistante
  • SyntaxError - Code invalide
  • ReferenceError - Variable non déclarée
  • NetworkError - Appels API échoués
  • Promise rejection - Promesse non catchée

3. Vérifier par navigateur/version

# Si erreur sur navigateur spécifique
# Vérifier compatibilité ES6/ES7
# Vérifier polyfills nécessaires

🛠️ Solutions

Erreurs de typage

// Vérifier l'existence avant d'utiliser
if (typeof obj.method === 'function') {
    obj.method();
}

// Optional chaining
const value = obj?.property?.method?.();

Erreurs réseau

// Gérer les erreurs fetch
fetch('/api/data')
    .then(response => {
        if (!response.ok) {
            throw new Error('HTTP error');
        }
        return response.json();
    })
    .catch(error => {
        console.error('API error:', error);
        showFallbackData();
    });

Problèmes de chargement

<!-- S'assurer que les scripts sont chargés -->
<script src="app.js" defer></script>
<script>
    window.addEventListener('load', () => {
        // Code dépendant du DOM
    });
</script>
