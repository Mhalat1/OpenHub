👁️ Runbook: Pages vues

Alerte: Anomalie dans le nombre de pages vues
Type: Monitoring trafic

📋 Description

Cette alerte se déclenche quand le volume de pages vues s'écarte significativement de la normale (hausse ou baisse anormale).

🔍 Investigation

Si baisse de trafic:

  • Vérifier si l'application répond (pas de 5xx)
  • Vérifier Google Search Console (problèmes SEO?)
  • Vérifier les campagnes marketing en cours
  • Vérifier si c'est un jour férié/weekend
  • Analyser par source de trafic

Si hausse de trafic:

  • Vérifier si capacité serveur tient
  • Identifier la source (campagne, viral, bot)
  • Si bot, mettre en place du rate limiting
  • Communiquer avec marketing si campagne

Analyser les logs

# Top URLs
awk '{print $7}' /var/log/nginx/access.log | sort | uniq -c | sort -rn | head -20

# Top referrers
awk -F'"' '{print $4}' /var/log/nginx/access.log | sort | uniq -c | sort -rn | head -10

# Top IPs
awk '{print $1}' /var/log/nginx/access.log | sort | uniq -c | sort -rn | head -20

🛠️ Actions

Si baisse:

1. Vérifier santé application
2. Vérifier SEO/indexation
3. Contacter marketing
4. Analyser concurrents

Si hausse (bonne):

1. Féliciter l'équipe marketing
2. Surveiller les ressources
3. Préparer scaling si nécessaire

Si hausse (bots):

# Ajouter rate limiting
# Laravel
Route::middleware('throttle:60,1')->group(...);

# Bloquer IPs suspectes
# Nginx
deny 192.168.1.1;
