⚡ 1. Actions immédiates (T0 - 5 minutes)

🔍 Étape 1: Vérifier l'état du service

# Tester l'API directement
curl -I https://open_hub-backend.onrender.com/health
curl -I https://open_hub-backend.onrender.com/api/v1/health

# Vérifier le status HTTP (doit retourner 200)
http_code=$(curl -s -o /dev/null -w "%{http_code}" https://open_hub-backend.onrender.com/health)
echo "HTTP Status: $http_code"

📊 Étape 2: Vérifier les métriques Grafana

# Accès direct
https://[votre-grafana]/d/backend-availability

# Métriques à vérifier:
- Taux de requêtes HTTP 5xx
- Temps de réponse moyen
- Utilisation CPU/Memory
- Connexions base de données

📝 Étape 3: Vérifier les logs

# Via Render Dashboard
https://dashboard.render.com/web/open_hub-backend/logs

# Via CLI (si configuré)
render logs --service open_hub-backend --tail 100 | grep -i "error\|exception\|panic"

# Rechercher les erreurs 5xx
grep "HTTP/1.1\" 5[0-9][0-9]" /var/log/access.log

🛠️ 3. Procédures de résolution

📦 Cas A: Redémarrage de l'application

# Via Render Dashboard
1. Aller sur https://dashboard.render.com
2. Sélectionner "open_hub-backend"
3. Cliquer sur "Manual Restart"
4. Attendre 30 secondes
5. Vérifier health endpoint

# Via Render CLI (si configuré)
render restart --service open_hub-backend
render ps --service open_hub-backend  # Vérifier status

💾 Cas B: Problème base de données

# Vérifier connexion DB
psql -h [db-host] -U [user] -d [db] -c "SELECT 1"

# Vérifier connections actives
psql -h [db-host] -U [user] -d [db] -c "SELECT count(*) FROM pg_stat_activity;"

# Killer les requêtes longues
psql -h [db-host] -U [user] -d [db] -c "SELECT pg_terminate_backend(pid) FROM pg_stat_activity WHERE state = 'active' AND now() - state_change > interval '5 minutes';"

🔄 Cas C: Rollback de déploiement

# Identifier la version stable
git log --oneline -10
# ou via Render Dashboard: Deployments → voir historique

# Effectuer rollback
# Option 1: Via Render Dashboard
Deploy → "Rollback" → Sélectionner version précédente

# Option 2: Via Git
git checkout [previous-stable-commit]
git push -f origin main

# Option 3: Via Render CLI
render deploy --version [commit-hash]

⚡ Cas D: Problème de ressources

# Vérifier utilisation actuelle
# Dashboard Render → Metrics

# Scale temporaire (si nécessaire)
# Render Dashboard → Settings → Instance Type
# Passer de "Starter" à "Professional" temporairement
