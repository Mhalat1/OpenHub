👆 Runbook: FID (First Input Delay) élevé

Alerte: First Input Delay > 100ms
Impact: Lenteur perçue par l'utilisateur

📋 Description

Le FID mesure le temps entre la première interaction utilisateur (clic, tap) et la réponse du navigateur. Un FID élevé indique que le thread principal est bloqué.

🔍 Causes

  • JavaScript long - Scripts qui bloquent le thread
  • Parsing CSS/HTML - Trop de travail au chargement
  • Exécution de tâches lourdes - Au démarrage
  • Trop de plugins - Analytics, chats, etc.

🛠️ Solutions

Optimiser JavaScript

// Découper le code en chunks
import('./heavyModule.js').then(module => {
    module.init();
});

// Reporter l'exécution
window.addEventListener('load', () => {
    // Code non critique
});

// Web Workers pour tâches lourdes
const worker = new Worker('worker.js');

Réduire le travail du thread principal

<!-- Charger JS en différé -->
<script src="app.js" defer></script>

<!-- CSS critique en inline -->
<style>
    /* CSS critique pour le premier rendu */
</style>
